### Context

Kasir One is positioned as a Web POS product for MSMEs, requiring a scalable foundation for cashier flow, product catalog, transactions, payments, customer/debt, and inventory movement.

### Solution

- Frontend foundation with file-based routing, typed API wrapper, and predictable server-state patterns using TanStack Query.
- Backend foundation with Better Auth session flows, RBAC guards (`authGuard`, `roleGuard`, `permissionGuard`), global rate limit, configurable CORS, and structured logging.
- POS schema prepared for stores, members, categories, products, transactions, transaction items, payment methods (CASH/QRIS/TRANSFER/CARD), customers/debts, and stock movements.

### Architecture Notes

- Frontend: alias-driven imports (`@/`), reusable layout/providers, and Bun-first tooling.
- Backend: middleware pipeline order (`logger` → `rate limit` → `cors` → `pretty json` → `auth middleware`) with route modules under `/api`.
- Deployment-ready path via local Bun workflows and Docker profiles for dev/prod.
