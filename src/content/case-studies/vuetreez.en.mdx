### Context

VueTreez was created as a reusable frontend starter to remove repetitive
boilerplate and give teams a stable baseline for modern Vue product development.

### Solution

- Pre-wired routing structure with `MainLayout` and dedicated Home/About/Not
  Found routes.
- Built-in internationalization with English and Indonesian locales and quick
  language switching.
- Theme switching (`light`, `dark`, `auto`) with persisted preference and
  predictable UI behavior.
- Persisted app state with Pinia + plugin-based storage and key prefix
  conventions.
- Public production deployment is available at `https://vuetreez.redwans.com/`.

### Architecture Notes

- CSS-first Tailwind v4 setup with design tokens defined in
  `src/assets/css/main.css`.
- Clean folder segmentation (`components`, `layouts`, `pages`, `router`,
  `stores`, `i18n`) to support maintainability.
- Bun-first scripts for install/dev/build/preview with Oxlint + Prettier for
  code quality.
